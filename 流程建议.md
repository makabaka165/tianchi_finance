# 流程建议

## 1. 当前比赛要做什么

这个赛题是一个典型的金融风控表格建模任务，目标是预测用户贷款是否违约。

训练集 `train.csv` 中有一列标签：

- `isDefault = 1`：用户违约
- `isDefault = 0`：用户未违约

测试集 `testA.csv` 没有 `isDefault`，需要模型预测每一行样本违约的概率。

最终提交文件格式参考 `sample_submit.csv`：

```csv
id,isDefault
800000,0.5
800001,0.5
800002,0.5
```

其中 `isDefault` 不是简单填 0 或 1，而是填模型预测出的“违约概率”，例如 `0.123456`。

比赛评测指标是 AUC，越高越好。AUC 关注的是模型对样本风险高低的排序能力，而不是单纯分类准确率。

## 2. 当前数据情况

根据当前文件初步查看：

- `train.csv`：约 80 万行，47 列
- `testA.csv`：约 20 万行，46 列
- `sample_submit.csv`：约 20 万行
- 训练集标签列：`isDefault`
- 训练集违约样本比例：约 19.95%
- 非违约样本比例：约 80.05%

这说明数据存在一定类别不均衡，但不算特别极端。建模时通常不需要特殊复杂处理，先用常规二分类模型即可。

字段类型大致包括：

- 数值特征：`loanAmnt`、`interestRate`、`installment`、`annualIncome`、`dti` 等
- 类别特征：`grade`、`subGrade`、`employmentLength`、`homeOwnership`、`purpose` 等
- 日期特征：`issueDate`、`earliesCreditLine`
- 匿名特征：`n0` 到 `n14`
- 高基数类别/编码特征：`employmentTitle`、`postCode`、`title`

有部分缺失值，例如：

- `n11` 缺失约 8.7%
- `employmentLength` 缺失约 5.8%
- 部分 `n` 系列匿名特征缺失约 5%

这类缺失值对 LightGBM、CatBoost 这类树模型比较友好，可以先不用特别复杂地填补。

## 3. 是微调模型还是训练模型

不建议微调大模型。

这个比赛不是文本生成、问答、图像识别或大语言模型任务，而是结构化表格二分类任务。最适合的方法是训练传统机器学习模型，尤其是梯度提升树模型。

推荐优先级：

1. LightGBM
2. CatBoost
3. XGBoost
4. Logistic Regression 作为简单对照

这里说的“训练模型”，不是从零训练一个神经网络大模型，而是用比赛数据训练一个表格分类器。训练成本和大模型微调完全不是一个量级。

最推荐先从 LightGBM 入手，因为它：

- 对表格数据效果很好
- 训练速度快
- 支持缺失值
- 对几十万到几百万行数据比较友好
- 比深度学习模型更适合新手快速出分

## 4. 当前电脑是否够用

你的电脑是 4060 Laptop，从这个赛题看，第一阶段完全够用。

需要注意的是，这个任务主要不是吃 GPU，而是更依赖：

- CPU
- 内存
- 磁盘读取速度
- 特征工程和验证策略

LightGBM 在 CPU 上就能跑得很好。对于当前 80 万行、几十列的数据规模，普通笔记本也能训练。

如果电脑内存有 16GB，通常可以先跑基础版本。如果有 32GB，会更舒服。

4060 Laptop GPU 对这个任务不是核心瓶颈。你可以后续尝试 CatBoost GPU 或 LightGBM GPU，但新手阶段没有必要一开始就折腾 GPU 配置。

## 5. 是否需要租额外算力平台

目前不建议一开始租算力。

建议先本地完成：

1. 跑通数据读取
2. 跑通基础特征处理
3. 跑通 LightGBM 训练
4. 生成可提交文件
5. 得到一个 baseline 分数

等你已经有 baseline 之后，如果遇到下面情况，再考虑租算力：

- 本地内存明显不够，经常崩溃
- 训练一次超过 1 小时，严重影响迭代
- 想同时跑很多组参数
- 想做多模型融合
- 想做大量特征工程实验
- 想跑自动调参

如果真要租，也不一定优先租 GPU。这个比赛更推荐租：

- CPU 核心更多
- 内存 32GB 或 64GB
- 磁盘 IO 更好的机器

GPU 不是第一优先级。

## 6. 新手推荐路线

建议分成四个阶段走。

### 阶段一：跑通 baseline

目标不是追求高分，而是先完整跑通比赛流程。

需要做的事情：

1. 读取 `train.csv`
2. 读取 `testA.csv`
3. 分离训练特征和标签
4. 做非常基础的特征处理
5. 使用 LightGBM 训练模型
6. 对 `testA.csv` 预测概率
7. 生成 `submission.csv`
8. 提交到比赛平台

第一版可以不追求复杂特征，只要能跑通，就已经完成了最重要的一步。

### 阶段二：做基础特征工程

可以优先处理这些字段：

#### `employmentLength`

原始值类似：

- `10+ years`
- `2 years`
- `< 1 year`
- `NaN`

可以转换成数字：

- `10+ years` 转成 `10`
- `2 years` 转成 `2`
- `< 1 year` 转成 `0`
- 缺失值保留或填 `-1`

#### `issueDate`

贷款发放日期，例如 `2014-07-01`。

可以转换出：

- 发放年份
- 发放月份
- 距离某个基准日期的月份数

#### `earliesCreditLine`

最早信用额度开立日期，例如 `Aug-2001`。

可以转换出：

- 最早授信年份
- 最早授信月份
- 信用历史长度

例如：

```text
信用历史长度 = 贷款发放日期 - 最早授信日期
```

#### `grade` 和 `subGrade`

贷款等级是很重要的特征。

可以：

- 用 Label Encoding 转成数字
- 或者让 CatBoost 原生处理类别特征

LightGBM 通常需要先编码。

#### `ficoRangeLow` 和 `ficoRangeHigh`

可以新增：

```text
ficoMean = (ficoRangeLow + ficoRangeHigh) / 2
```

#### 贷款收入相关比例

可以新增：

```text
loanIncomeRatio = loanAmnt / annualIncome
installmentIncomeRatio = installment * 12 / annualIncome
```

这些特征直觉上和还款压力有关。

### 阶段三：交叉验证

不要只切一次训练集，否则分数可能不稳定。

推荐使用 5 折 StratifiedKFold：

1. 把 80 万训练数据分成 5 份
2. 每次用 4 份训练，1 份验证
3. 重复 5 次
4. 记录每折 AUC
5. 取 5 折模型对测试集预测结果的平均值

这样做的好处：

- 更稳定
- 更接近线上分数
- 能判断特征是否真的有效
- 生成的测试集预测通常比单模型更好

### 阶段四：优化和融合

baseline 跑通之后再考虑：

- 调 LightGBM 参数
- 加更多组合特征
- 尝试 CatBoost
- 尝试 XGBoost
- 多模型平均融合
- 用不同随机种子训练多个模型再平均

新手不建议一开始就做复杂融合。先把单模型做到稳定，再逐步加复杂度。

## 7. 建议的实际操作流程

下面是推荐操作顺序，不代表现在必须执行。

### 第一步：准备 Python 环境

建议使用单独虚拟环境，避免污染系统 Python。

可以安装这些库：

```bash
pip install pandas numpy scikit-learn lightgbm
```

如果安装 LightGBM 报错，可以先只安装：

```bash
pip install pandas numpy scikit-learn
```

然后再单独解决 LightGBM。

### 第二步：先写一个 baseline 脚本

建议命名为：

```text
baseline_lgb.py
```

这个脚本负责：

- 读取数据
- 处理简单特征
- Label Encoding
- 训练 LightGBM
- 计算本地 AUC
- 输出提交文件

### 第三步：先少量参数训练

第一版参数不要太复杂，例如：

```text
learning_rate = 0.05
num_leaves = 64
n_estimators = 2000
subsample = 0.8
colsample_bytree = 0.8
```

使用 early stopping，让模型自己在验证集效果不提升时停止。

### 第四步：生成提交文件

输出文件建议命名为：

```text
submission_lgb_baseline.csv
```

格式必须是：

```csv
id,isDefault
800000,0.123456
800001,0.234567
```

注意：

- `id` 必须来自 `testA.csv`
- `isDefault` 是概率
- 不要打乱测试集顺序
- 列名必须和样例一致

### 第五步：提交并记录结果

每次提交后建议记录：

```text
日期
使用模型
主要特征
本地 CV AUC
线上 AUC
备注
```

建议新建一个实验记录表，例如：

```text
实验记录.md
```

这样后面不会忘记哪次改动有效。

## 8. 推荐的优先实现方案

如果你是新手，推荐第一版这样做：

模型：

```text
LightGBM
```

验证：

```text
5 折 StratifiedKFold
```

特征处理：

```text
保留大部分原始字段
删除 id
转换 employmentLength
转换 issueDate
转换 earliesCreditLine
Label Encoding 类别字段
新增少量比例特征
```

暂时不做：

```text
深度学习
大模型微调
复杂神经网络
自动机器学习
过度特征筛选
复杂模型融合
```

## 9. 新手容易踩的坑

### 不要用准确率作为主要指标

这个比赛看 AUC，不是 accuracy。

由于非违约样本约 80%，如果模型全部预测不违约，准确率也可能看起来不低，但 AUC 和比赛成绩不会好。

### 不要把测试集标签当成已知

`testA.csv` 没有 `isDefault`，不能自己造标签训练。

### 不要把 `id` 当作普通特征

`id` 通常只是样本编号。第一版建议不要作为训练特征。

### 注意训练集和测试集一起编码

如果对类别特征做 Label Encoding，建议把训练集和测试集拼在一起 fit 编码器，再分别拆开。

否则测试集中可能出现训练集中没有的类别，导致报错。

### 不要一开始就追求复杂模型

这个比赛的关键通常是：

```text
稳定验证 + 合理特征 + LightGBM/CatBoost 参数 + 融合
```

而不是大模型或复杂神经网络。

### 不要频繁盲目提交

先看本地交叉验证 AUC 是否提升，再决定是否提交。

## 10. 推荐决策

综合当前数据、任务类型和你的电脑配置，建议如下：

1. 不做大模型微调。
2. 不从零训练深度学习模型。
3. 不急着租算力。
4. 先本地跑 LightGBM baseline。
5. 跑通后再逐步做特征工程和调参。
6. 如果后期实验变多，再考虑租 CPU/内存更强的机器。

最合理的第一目标不是马上拿高分，而是：

```text
先生成一个格式正确、能提交、有本地 AUC 的 baseline。
```

完成这一步后，再继续优化会清晰很多。
