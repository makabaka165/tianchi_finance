## 实验 001：LightGBM 基础 Baseline

### 时间

2026-05-09

### 代码变更

新增文件：

- `baseline_lgb.py`
- `实验记录.md`

### 主要目标

建立第一版可复现的 baseline 流程：

1. 读取 `train.csv` 和 `testA.csv`
2. 做基础特征工程
3. 使用 LightGBM 训练二分类模型
4. 使用 5 折 StratifiedKFold 做本地验证
5. 输出测试集预测提交文件
6. 输出 OOF 验证预测和特征重要性，方便后续分析

### 算法重点

模型：

```text
LightGBM GBDT 二分类模型
```

验证方式：

```text
5 折 StratifiedKFold
```

评价指标：

```text
AUC
```

基础特征处理：

- `employmentLength` 转换为数字年限
- `issueDate` 拆分为年份、月份、月份序号
- `earliesCreditLine` 拆分为最早授信年份、月份
- 构造信用历史长度
- 构造 FICO 均值和区间宽度
- 构造贷款金额 / 年收入、年还款额 / 年收入等比例特征
- 对 `grade`、`subGrade` 等类别字段做 Label Encoding
- 对 `employmentTitle`、`postCode`、`title`、`regionCode`、`purpose` 做计数特征


完整训练：

### 结论

第一版 LightGBM baseline 已经跑通，输出文件格式正确，可以作为第一份可提交结果。

本地 5 折 OOF AUC 为 0.737236，属于一个合理的起步分数。后续优化重点建议：

1. 查看特征重要性，确认哪些字段贡献最大。
2. 增强日期特征、收入/负债比例特征、类别统计特征。
3. 尝试 CatBoost，尤其是让模型更自然地处理类别特征。
4. 调整 LightGBM 参数，例如 `num_leaves`、`min_child_samples`、`learning_rate`、`reg_alpha`、`reg_lambda`。
5. 后续如果做模型融合，保留当前结果作为 baseline 对照。

## 实验 002：基于特征重要性的 LightGBM 特征增强

### 时间

2026-05-09

### 代码变更

修改文件：

- `baseline_lgb.py`
- `实验记录.md`

### 主要目标

在不改变模型类型和验证方式的前提下，基于实验 001 的特征重要性做一版低风险特征增强。

baseline 对照：

```text
实验 001 OOF AUC = 0.737236
```

### 特征重要性观察

实验 001 中排名靠前的特征包括：

- `issue_month_index`
- `dti`
- `revolUtil`
- `annualIncome`
- `revolbal_income_ratio`
- `openacc_totalacc_ratio`
- `loan_income_ratio`
- `interestRate`
- `credit_history_months`
- `postCode_count`
- `installment_income_ratio`
- `employmentTitle`
- `regionCode_count`

这说明当前模型主要依赖时间趋势、负债收入比、额度利用率、收入/还款压力、信用历史长度、高基数字段频次等信息。

### 算法重点

模型和验证方式保持不变：

```text
LightGBM + 5 折 StratifiedKFold + AUC
```

计划新增特征方向：

- 时间分段特征：季度、半年、信用历史年数等
- 贷款等级显式序号：`grade_rank`、`subgrade_rank`
- 还款压力交叉特征：利率、期限、贷款收入比、分期收入比、`dti` 的组合
- 匿名 `n0` 到 `n14` 聚合特征：缺失数量、总和、均值、最大值、标准差
- 高基数字段分组统计：按 `postCode`、`regionCode`、`purpose`、`grade`、`subGrade` 统计贷款金额、收入、利率、`dti` 等均值

实验 002 相比 baseline 有小幅提升，幅度为 `+0.000264`。这个提升是真实但不够大，暂时不建议占用官网提交次数。

- 本地 5 折 OOF AUC 有提升。
- 新增特征逻辑可解释。
- 输出预测分布正常。
- 新增特征在重要性中确实靠前。

但当前还没有达到官网提交门槛：

```text

```

下一步建议：

1. 继续在实验 002 基础上做更有针对性的特征筛选，避免特征数量增加但收益有限。
2. 尝试更小范围的 LightGBM 参数调整，例如 `num_leaves`、`min_child_samples`、`reg_lambda`。
3. 后续可以新增 CatBoost 单模型作为差异化对照。
