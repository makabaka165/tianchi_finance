## 实验 001：LightGBM 基础 Baseline

### 时间

2026-05-09

### 代码变更

新增文件：

- `baseline_lgb.py`
- `实验记录.md`

### 主要目标

建立第一版可复现的 baseline 流程：

1. 读取 `train.csv` 和 `testA.csv`
2. 做基础特征工程
3. 使用 LightGBM 训练二分类模型
4. 使用 5 折 StratifiedKFold 做本地验证
5. 输出测试集预测提交文件

```text

```

验证方式：

```text

```

评价指标：

```text

```

基础特征处理：

- `employmentLength` 转换为数字年限
- `issueDate` 拆分为年份、月份、月份序号
- `earliesCreditLine` 拆分为最早授信年份、月份

- 构造 FICO 均值和区间宽度

- 对 `grade`、`subGrade` 等类别字段做 Label Encoding
- 对 `employmentTitle`、`postCode`、`title`、`regionCode`、`purpose` 做计数特征

### 主要目标

在实验 002 的特征基础上，加入交叉验证安全的目标编码特征，尝试提升高基数类别字段的信息表达。

对照结果：

```text
实验 001 OOF AUC = 0.737236
实验 002 OOF AUC = 0.737499
```

### 算法重点

目标编码字段：

- `grade`
- `subGrade`
- `employmentTitle`
- `postCode`
- `regionCode`
- `purpose`
- `title`
- `homeOwnership`
- `verificationStatus`

目标编码方式：

- 在每个外层验证 fold 内，只使用该 fold 的训练部分计算编码映射。
- 外层验证集不参与自身编码统计，避免验证集标签泄漏。
- 外层训练部分内部再做 inner StratifiedKFold，生成 out-of-fold target mean。
- 测试集在每个外层 fold 中，只使用该 fold 训练部分计算出来的映射。
- 使用全局违约率做平滑，降低小类别过拟合风险。

运行参数：

```text
--target-encoding
--target-encoding-splits 5
--target-encoding-smoothing 20
```
